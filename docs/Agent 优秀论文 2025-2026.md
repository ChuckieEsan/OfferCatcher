## 一、先给结论：2025–2026 的三条主线

### 1. 记忆模块（Memory）正在从“外挂缓存”升级为 Agent 的核心系统层
2025 年的重点还是 **如何让 Agent 在测试时持续积累经验**，到 2026 年，研究已经明显转向：
- 如何把 memory 形式化为 **write–manage–read loop**
- 如何评测 memory 的真实收益，而不是只看 long-context QA
- 如何处理 **选择性遗忘、长期状态、轨迹压缩、经验重用**

### 2. 上下文工程（Context Engineering）正在取代单纯 Prompt Engineering
2025 下半年开始，社区明显从“提示词怎么写”转向“**整个上下文状态如何组织、裁剪、更新、检索、传递**”。
这条线与 Agent 的长期执行能力、工具调用成本、轨迹质量、MCP 接入方式强相关。

### 3. Harness Engineering 在 2026 年开始被明确命名并走向“可研究对象”
此前大量 Agent 性能提升，其实来自 harness（控制逻辑、状态管理、环境搭建、反馈回路、评测与修复），但这部分经常埋在工程代码里。2026 年出现了一批更明确把 **harness 本身** 当成研究与优化对象的工作。

- **类型**：论文

- **链接**：https://arxiv.org/abs/2504.07952
- **为什么重要**：这篇工作非常关键，因为它把“记忆”从静态存储，推进到 **测试时持续更新的经验手册（adaptive cheatsheet）**。核心思想不是简单保留历史对话，而是把解题策略、经验、错误模式沉淀成可复用知识。

## 四、研究动态判断（给你写综述或选题时可直接引用）

### 1. Memory 与 Context Engineering 正在合流
到 2025 年底、2026 年初，越来越多工作不再把 memory 和 context 当成两个完全分离的模块，而是把 memory 看成 **上下文构建与更新机制的一部分**。

### 2. Agent 的性能瓶颈越来越像“系统设计问题”，不只是模型能力问题
Harness engineering、MCP、code execution、state persistence、tool loading、eval-driven iteration，这些都表明：Agent 的上限越来越取决于 **外部 scaffold**。

### 3. Harness 正在从“隐性工程经验”变成“显性研究对象”
2026 年的 NLAHs 和 Meta-Harness 很值得持续跟踪。它们意味着未来可能会出现：
- harness benchmark
- harness search / harness optimization
- harness portability
- harness representation learning

### 4. 评测将成为 memory / context / harness 三条线的共同核心
接下来值得重点关注的问题包括：
- memory 是否真的提升长期任务成功率？
- context engineering 是否改善而不是污染行为？
- harness 优化是否会引入过拟合到评测集的问题？
